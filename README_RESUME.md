# 项目：企业级多 Agent 智能客服系统

## 项目背景

传统客服系统面临三大痛点：① 人工客服成本高、响应慢，日均处理量有限；② 知识库检索准确率不稳定，长尾问题覆盖不足；③ 缺乏情绪感知和兜底机制，复杂场景下用户体验差。本项目基于 LangGraph + ReAct 框架，构建了一套支持多 Agent 协作、多渠道接入、智能转人工的企业级智能客服系统，覆盖售前咨询、售后排查、投诉处理等全场景。

## 项目职责

### 1. 多 Agent 架构设计与实现
- 设计并实现 **6 个独立 Agent 子图**（FAQ Agent / RAG Agent / Reflect Agent / Expert Delegate / Human Coordinator / CustomerService Agent），基于 LangGraph StateGraph 编排，支持按意图自动路由到对应模型
- 实现 **多模型路由层**，按任务复杂度分配算力：Lite（qwen-turbo/7B）处理 FAQ、Medium（qwen-plus/34B）处理 RAG 排查、Heavy（qwen-max/70B）处理专家委托、Safety 专用模型处理质检，实现算力弹性分配
- 设计 **A2A 专家委托机制**，客服 Agent 遇到超纲问题时通过 Agent-to-Agent 协议远程委托给专家 Agent，支持超时回退到人工转接

### 2. RAG 检索系统优化
- 实现 **混合检索架构**：向量检索（ChromaDB）+ BM25 关键词检索 + RRF 融合排序，双索引（段落级 + 句子级）并行召回，句子级结果自动展开上下文窗口
- 设计 **三级权限过滤**：租户隔离 → 权限等级过滤 → 版本冲突检测，确保不同订阅计划用户只能访问对应级别的知识文档
- 实现 **幻觉防护机制**：检索置信度检查（token 密度阈值）+ LLM-as-Judge 交叉验证 + ReAct 中间步骤证据支撑检查，降低编造率

### 3. 对话状态机与意图管理
- 设计 **四级对话状态机**（INIT → COLLECTING → CONFIRMING → EXECUTING → RESOLVED/ESCALATED），管理槽位收集、用户确认、执行操作的完整生命周期
- 实现 **意图澄清机制**：检测缺失信息（错误码/产品范围/技术环境），优先从长期记忆中推断，推断失败时生成定向追问，避免无效多轮对话
- 集成 **情绪感知引擎**：基于关键词 + 上下文的愤怒/紧急/不满检测，检测到投诉关键词（"投诉""骗钱""报警"等）直接跳过追问转人工，检测到合规敏感词（"起诉""消协""12315"）标记为 critical 紧急度

### 4. 实时通信与人工转接
- 实现 **WebSocket 实时通信层**：用户端 `/ws/chat` 支持流式输出（typing_indicator → streaming_chunk → done）、心跳保活、自动重连；坐席端 `/ws/agent/{id}` 支持坐席登录/登出/回复
- 设计 **完整转接上下文包**：对话摘要 + 用户画像 + 已尝试方案 + 紧急度评估 + 卡点分析 + 完整对话记录，人工坐席可一目了然了解 AI 未能解决的问题
- 实现 **转接分发器**：支持在线坐席轮询分配、坐席不可用时的排队兜底、Copilot 建议回复自动推送

### 5. 多渠道接入与消息归一化
- 实现 **微信渠道接入**：公众号回调消息处理（文本/图片/语音/事件推送）、签名验证、菜单事件路由、XML 回复构建
- 实现 **电话 IVR 接入**：Twilio Webhook 集成、ASR 语音转文字、TTS 文本转语音、消息格式清理
- 设计 **消息归一化层**：统一微信/电话/网页/APP 等不同渠道的消息格式（user_id, channel, content, priority），屏蔽渠道差异

### 6. 效果评估与监控体系
- 实现 **四维度效果评估埋点**：业务指标（解决率/转人工率/平均轮数）、质量指标（平均质量分）、风险指标（幻觉率）、系统指标（工具调用成功率）
- 设计 **QualityReflector 质检 Agent**：对 Agent 回复进行四维审查（事实准确性/完整性/安全性/证据支撑），发现问题时自动修正回复
- 实现 **监控 API**：`/api/v1/metrics/*` 暴露实时指标，支持 Prometheus 集成

## 项目亮点

### 检索能力
- **混合检索**：向量 + BM25 + RRF 融合，双索引（段落级 + 句子级）并行召回，句子命中时自动展开前后 3 句上下文
- **版本冲突检测**：同一文档多版本自动排序 + 废弃过滤 + 活跃版本冲突提示
- **权限感知检索**：三级过滤（租户隔离 → 权限等级 → 版本冲突），支持 public/internal/confidential/restricted 四级权限

### 多 Agent 协作
- **6 个独立子图**：每个 Agent 可独立编译、独立扩展，FAQ Agent 零 LLM 调用（纯关键词匹配），Reflect Agent 使用专用安全模型
- **模型路由**：Lite/Medium/Heavy/Safety 四档模型按任务复杂度自动分配，FAQ 走 7B 轻量模型，RAG 走 34B 中量模型，质检走安全专用模型
- **A2A 专家委托**：客服 Agent 遇到超纲问题时通过 A2A 协议远程委托，支持超时回退

### 兜底与风控
- **三级兜底机制**：情绪检测（愤怒→直转人工）→ 澄清失败（追问超限→转人工）→ 置信度不足（score<0.3→拒答转人工）
- **注入攻击拦截**：entry_node 层拦截 prompt injection，拒绝执行危险指令
- **权限版本追踪**：多工具任务中途权限变更自动检测，已执行动作记录补偿状态（rollback/retry/manual_review）

## 项目成果

- **代码规模**：102 个 Python 源文件，18,322 行代码，覆盖 17 个模块
- **测试覆盖**：74 个测试用例，67 个通过（4 个预存失败 + 10 个预存错误为 API 不可用导致，非本次改动引入）
- **架构分层**：四层架构（接入层/调度中枢/专业 Agent/底层支撑），模块间松耦合
- **多渠道支持**：HTTP/WS 网页 + 微信 + 电话 IVR + 工单表单，统一消息归一化
- **Mock/Real 切换**：订单/CRM/工单/物流四大系统适配器支持一键切换（`ADAPTERS_USE_REAL=true`），开发用 Mock，生产用真实 API
